# 现象
- 客户使用 FTP 连接到服务器。在这台服务器上，可以获得配方文件。
- 可以实现的效果：
    - 可以连接到服务器  
    - 目录信息有效。  
    - 可以在那里创建一个具有可选名称的文件。  

- 但是，您无法打开文件并向其中写入内容。或重命名它。它总是写该文件不存在 - 错误 20708。 

# 原因
- 问题是服务器是Linux操作系统，Linux的默认文件系统不是 FAT32，而是ext4。  
  
  
# 解决办法
- 解决方案是不打开文件。
- DirInfo会发现那里有什么文件。
- 可以通过FileCopy 将文件复制到 PLC 上的 F 分区。您可以在此处打开文件并使用它。
